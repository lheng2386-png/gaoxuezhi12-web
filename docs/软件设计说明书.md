# 高血脂风险分层预警与干预推荐系统 Web版 V1.0 软件设计说明书

## 1. 设计目标

本系统旨在将中西医融合高血脂风险识别、三级分层预警与个体化干预优化模型实现为可运行的 Web 应用软件。系统强调操作简洁、结果可解释、部署轻量和模块结构清晰，便于科研展示、教学演示和软件著作权登记。

## 2. 总体架构

系统采用 Web 单服务架构。Flask 后端同时负责静态页面分发和 REST API 服务，前端通过 JavaScript 调用后端接口完成计算。

```text
浏览器页面
  |
  | HTTP 请求
  v
Flask Web 服务
  |
  +-- 前端页面: frontend/index.html, frontend/about.html
  |
  +-- API 路由: /api, /api/predict, /api/thresholds, /api/alternatives
  |
  +-- 模型模块:
      +-- 风险分层模型
      +-- 患者画像分型模型
      +-- 干预组合优化模型
      +-- 可选关键因素识别模型
```

## 3. 模块划分

### 3.1 前端交互模块

文件位置：`frontend/index.html`

主要职责：

- 展示软件首页和操作流程
- 收集用户输入
- 提供低风险、中风险和高风险示例病例填充
- 调用后端预测接口
- 展示风险等级、异常因素、患者画像、指标阈值图和干预方案
- 生成评估报告，支持复制和打印
- 使用浏览器本地存储保存最近评估历史
- 提供重新评估和步骤切换功能

### 3.2 Web 服务与 API 模块

文件位置：`backend/app.py`

主要职责：

- 提供 `/` Web 首页
- 提供 `/about` 关于页面
- 提供 `/api` 服务信息接口
- 提供 `/api/predict` 完整预测接口
- 提供 `/api/thresholds` 阈值查询接口
- 提供 `/api/alternatives` 备选方案接口
- 支持通过环境变量 `PORT` 指定端口

### 3.3 风险分层模型模块

文件位置：`backend/models/risk_classification.py`

主要职责：

- 加载风险阈值配置
- 根据痰湿积分、血脂指标、BMI、血尿酸和活动能力评分判断异常因素
- 将风险分数映射为低、中、高三级风险
- 返回风险等级、颜色、描述和异常因素列表

### 3.4 干预优化模型模块

文件位置：`backend/models/intervention_optimization.py`

主要职责：

- 根据用户特征完成患者画像分型
- 根据患者类型设定约束条件
- 枚举调理等级和运动方案组合
- 在预算约束下选择最优方案
- 返回最优方案、备选方案、成本和预期效果

### 3.5 关键因素识别模块

文件位置：`backend/models/feature_identification.py`

主要职责：

- 使用 Lasso-Logistic 进行特征筛选
- 使用 XGBoost-SHAP 计算非线性特征重要性
- 使用随机森林进行交叉验证
- 输出多模型共识核心特征

该模块为可选训练型接口，普通 Web 风险评估不依赖其重型机器学习环境。

### 3.6 报告与历史记录模块

文件位置：`frontend/index.html`

主要职责：

- 将评估结果组织为报告预览
- 通过浏览器打印能力输出报告
- 将最近评估记录保存到 `localStorage`
- 支持从历史记录恢复风险结果
- 支持清空本地历史记录

## 4. 数据设计

系统当前版本不使用数据库，主要数据包括用户输入数据、阈值配置数据和干预参数配置数据。

### 4.1 用户输入数据

| 字段 | 含义 |
|------|------|
| age | 年龄 |
| gender | 性别 |
| height | 身高 |
| weight | 体重 |
| tan_score | 痰湿体质积分 |
| tg | 甘油三酯 |
| tc | 总胆固醇 |
| hdl_c | 高密度脂蛋白 |
| ldl_c | 低密度脂蛋白 |
| urea_acid | 血尿酸 |
| adl_score | 活动能力评分 |
| budget | 6 个月预算上限 |

### 4.2 配置文件

| 文件 | 说明 |
|------|------|
| backend/config/risk_thresholds.json | 风险分层阈值 |
| backend/config/intervention_costs.json | 干预成本、效果和患者类型约束 |

## 5. 接口设计

### 5.1 `/api`

请求方式：GET

功能：返回系统名称、版本、模块和论文来源信息。

### 5.2 `/api/predict`

请求方式：POST

功能：完成风险评估和干预推荐。

输入示例：

```json
{
  "age": 62,
  "gender": 1,
  "height": 170,
  "weight": 82,
  "tan_score": 66,
  "tg": 2.3,
  "tc": 6.5,
  "hdl_c": 0.9,
  "ldl_c": 4.3,
  "urea_acid": 450,
  "adl_score": 38,
  "budget": 2000
}
```

输出内容包括：

- 输入摘要
- 风险分层结果
- 患者画像分型
- 最优干预方案
- 备选干预方案

### 5.3 `/api/thresholds`

请求方式：GET

功能：返回系统当前使用的风险阈值配置。

### 5.4 `/api/alternatives`

请求方式：POST

功能：根据患者类型和预算返回备选干预方案。

## 6. 算法设计

### 6.1 三级风险分层算法

系统首先按照阈值对指标进行判断。对于明显异常指标赋予较高风险权重，对于轻度异常指标赋予较低风险权重，再根据总风险分数输出三级风险。

风险等级映射：

- 低风险：风险分数较低
- 中风险：存在部分异常因素
- 高风险：存在多项明显异常因素

### 6.2 患者画像分型算法

系统按照活动能力、年龄、BMI 和痰湿体质积分进行规则判断：

- 活动能力低或高龄：优先判定为功能受限型
- BMI 超标且痰湿积分偏高：判定为肥胖并发型
- 其他血脂异常为主的情况：判定为代谢异常主导型

### 6.3 干预组合优化算法

系统将调理等级和运动方案视为离散决策变量，在预算约束下枚举全部可行方案。每个方案包含成本和预期效果，系统优先选择效果最高的方案；若效果接近，则优先选择成本更低的方案。

## 7. 安全与异常处理

- 对必填字段进行缺失检查
- 对接口异常返回统一 JSON 错误信息
- 前端对 API 调用失败进行提示
- 系统不保存个人健康数据，降低隐私存储风险

## 8. 部署设计

本系统支持本地部署和普通 Web 平台部署。通过 `PORT` 环境变量可适配不同运行环境。

默认运行：

```bash
python backend/app.py
```

指定端口：

```bash
PORT=5050 python backend/app.py
```

## 9. 维护说明

- 风险阈值可在 `backend/config/risk_thresholds.json` 中维护
- 干预成本和效果参数可在 `backend/config/intervention_costs.json` 中维护
- 前端页面文案和交互可在 `frontend/index.html` 中维护
- API 逻辑可在 `backend/app.py` 中维护
