# 高血脂风险分层预警与干预推荐系统 Web版 V1.0 用户使用手册

## 1. 软件简介

高血脂风险分层预警与干预推荐系统 Web版 V1.0 是一款面向中老年健康管理场景的 Web 应用软件。系统融合中医体质积分、活动能力评分、血脂指标和相关生理特征，完成高血脂风险辅助识别、三级分层预警、患者画像分型和个体化干预方案推荐。

本软件适用于科研、教学、社区健康管理和软件著作权登记演示等场景，不作为临床诊断或治疗依据。

## 2. 运行环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows、macOS、Linux |
| Python | 3.8 或以上 |
| 浏览器 | Chrome、Edge、Safari 等现代浏览器 |
| 后端框架 | Flask |
| 前端技术 | HTML5、Bootstrap 5、JavaScript |

## 3. 安装与启动

### 3.1 安装依赖

进入项目根目录后执行：

```bash
pip install -r requirements.txt
```

### 3.2 启动服务

默认端口启动：

```bash
python backend/app.py
```

指定端口启动：

```bash
PORT=5050 python backend/app.py
```

### 3.3 访问系统

默认访问地址：

```text
http://127.0.0.1:5000
```

如果使用 5050 端口，则访问：

```text
http://127.0.0.1:5050
```

## 4. 操作流程

### 4.1 打开首页

用户在浏览器中打开系统地址后，进入信息录入页面。页面顶部显示系统名称、版本和研究来源，主体区域按照步骤展示操作流程。

### 4.2 使用示例病例

系统侧边栏和表单底部提供低风险、中风险、高风险三个示例病例按钮。用户可点击任一示例，系统会自动填充对应数据，便于快速演示和测试。

### 4.3 填写基本信息

用户需要填写以下基本信息：

- 年龄
- 性别
- 身高
- 体重

系统根据身高和体重自动计算 BMI，并将 BMI 作为风险分层和患者画像分型的重要指标。

### 4.4 填写生化指标

用户填写最近一次体检中的血脂和代谢相关指标：

- 甘油三酯 TG
- 总胆固醇 TC
- 高密度脂蛋白 HDL-C
- 低密度脂蛋白 LDL-C
- 血尿酸

系统会按照内置阈值判断是否存在偏高、过高或偏低等异常情况。

### 4.5 填写中医体质与活动能力信息

用户填写：

- 痰湿体质积分，取值范围 0 至 100 分
- ADL 活动能力评分，取值范围 0 至 100 分

痰湿体质积分越高，提示痰湿偏颇越明显；活动能力评分越低，提示活动受限越明显。

### 4.6 填写干预预算

用户填写 6 个月干预预算上限。系统默认预算为 2000 元，并在该预算约束下筛选可行干预组合。

### 4.7 执行风险评估

用户点击“下一步：计算风险”后，系统调用后端 API 执行以下计算：

1. 判断异常风险因素
2. 进行低、中、高三级风险分层
3. 完成患者画像分型
4. 生成个体化干预方案

### 4.8 查看风险结果

系统显示风险等级，包括：

- 低风险
- 中风险
- 高风险

同时展示异常风险因素列表，例如痰湿体质积分过高、甘油三酯过高、活动能力明显受限等。

系统还会展示指标与阈值对比图，红色条形表示达到风险阈值或低于保护性指标阈值。

### 4.9 查看患者画像

系统根据用户特征自动给出患者画像类型：

- 代谢异常主导型
- 肥胖并发型
- 功能受限型

页面展示该类型的说明、推荐策略和论文结论摘要。

### 4.10 查看干预推荐

系统展示预算约束下的最优干预方案，包括：

- 调理方案名称与适用痰湿积分范围
- 调理方案内容
- 活动干预强度
- 每周训练频率
- 单次训练时长
- 单次训练成本
- 6 个月调理费用、运动费用和总成本
- 预期痰湿积分下降比例
- 年龄、活动能力和预算约束依据

系统还会展示其他备选方案，供用户进行比较。

### 4.11 生成和打印报告

进入报告页后，系统会自动生成评估报告。用户可以复制报告文本，也可以使用打印功能输出纸质版或保存为 PDF。

## 5. 输出结果说明

### 5.1 风险等级

| 风险等级 | 含义 |
|----------|------|
| 低风险 | 指标基本正常，建议维持健康生活方式 |
| 中风险 | 部分指标异常，建议关注并调整生活方式 |
| 高风险 | 多项指标异常，建议及时咨询专业医师 |

### 5.2 异常因素

异常因素用于解释风险等级来源，便于用户理解风险判断依据。

### 5.3 干预方案

干预方案由中医调理等级和活动干预方案组合构成。系统依据原题附表2、附表3匹配调理等级，按年龄和活动量表评分限制活动强度，并在6个月预算内筛选成本合理、预期痰湿积分下降较高的方案。

## 6. 常见问题

### 6.1 页面无法打开怎么办？

请确认后端服务已启动，并检查访问端口是否正确。如果 5000 端口被占用，可以使用 `PORT=5050 python backend/app.py` 启动服务。

### 6.2 计算失败怎么办？

请确认所有必填项均已填写，且数值在合理范围内。也可以检查浏览器页面中的 API 地址是否为 `/api`。

### 6.3 是否需要数据库？

当前版本无需数据库，系统使用内置 JSON 配置文件保存阈值和干预参数。

### 6.4 是否可以重新训练模型？

普通风险评估不需要重新训练。如需使用关键因素识别训练接口，可额外安装 `backend/requirements-ml.txt` 中的机器学习依赖。
