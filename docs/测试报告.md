# 高血脂风险分层预警与干预推荐系统 Web版 V1.0 测试报告

## 1. 测试目的

验证系统在 Web 运行环境下能否正常启动、访问、完成风险评估、输出患者画像和生成干预推荐方案，确保软件满足基本使用和软件著作权登记演示要求。

## 2. 测试环境

| 项目 | 内容 |
|------|------|
| 操作系统 | macOS / Windows / Linux 均可 |
| Python | 3.8+ |
| 浏览器 | Chrome、Edge、Safari |
| 后端框架 | Flask |
| 测试地址 | http://127.0.0.1:5050 |

## 3. 测试范围

- Web 首页访问
- API 服务信息接口
- 风险评估接口
- 低风险、中风险、高风险样例
- 患者画像分型
- 干预方案推荐
- 缺失字段异常处理
- 端口配置

### 用例 3：高风险样例

输入：

```json
{
  "age": 62,
  "gender": 1,
  "height": 170,
  "weight": 82,
  "tan_score": 66,
  "tg": 2.3,
  "tc": 6.5,
  "hdl_c": 0.9,
  "ldl_c": 4.3,
  "urea_acid": 450,
  "adl_score": 38,
  "budget": 2000
}
```

预期结果：

- 返回 code 为 0
- 风险等级为高风险
- 患者类型为功能受限型
- 返回最优干预方案和 3 个备选方案

实际结果：通过。
