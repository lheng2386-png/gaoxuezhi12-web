# 高血脂风险分层预警与干预推荐系统 Web版 - 前端

## 说明

前端采用简单的 HTML + Bootstrap 5 + 原生 JavaScript 实现，满足：

1. 四个核心页面：信息录入 → 风险计算 → 风险结果 → 干预推荐
2. 前后端分离代码结构，通过 REST API 与后端通信
3. 响应式设计，适配桌面浏览器和常见移动端宽度
4. 由 Flask 单服务统一提供页面和 API，适合软著演示和本地部署

## 文件结构

```
frontend/
├── index.html       # 主页面（包含四个步骤）
├── about.html       # 关于页面
└── README.md        # 说明文件
```

## 使用方法

1. 在项目根目录安装依赖：`pip install -r requirements.txt`
2. 启动服务：`python backend/app.py`
3. 浏览器访问：`http://127.0.0.1:5000`
4. 页面默认 API 地址为 `/api`，通常无需修改
5. 填写用户信息，点击下一步即可完成评估

## 页面流程

| 步骤 | 页面 | 功能 |
|------|------|------|
| 1 | 信息录入 | 输入年龄、性别、身高、体重、血脂指标、痰湿积分、活动能力、预算 |
| 2 | 风险计算 | 调用后端API，异步计算结果 |
| 3 | 风险结果 | 展示风险等级、异常因素、患者分型 |
| 4 | 干预推荐 | 展示最优方案和备选方案，包括成本与预期效果 |

## 前后端接口

前端调用 `POST /api/predict` 接口，传入用户信息，后端返回完整结果。
